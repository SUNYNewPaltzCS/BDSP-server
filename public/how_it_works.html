How It Works
System Map

Data collection, performance metrics,
evaluation, and analysis of the real world impacts
of administrative decisions

The Server

The server is the brains of the operation.

Initially the server is used to create a fully customized Android application for data collection that is tailored specifically to the client's needs.

The App

Data collection tools are tailored to your needs.

The application is capable of taking voice to text input, log GPS locations and routes, take pictures, and also has the capability to pull data from compatible environmental sensors via Bluetooth. Once the Android application is generated, it is loaded onto the data collection devices.

The Collection Method

In the field or on the move, capture the data you need.

Inexpensive Android phones with custom operating systems are then sent into the field to collect the data. Some data, such as GPS routes, is collected automatically. Other data is inputted by people in the field.

Data Collected

From anywhere to your database

The data collected in the field is stored in a database on the phone. This makes it possible to collect and store data using the system in areas where there is no cell service, and without incurring the cost of a mobile plan with a cell phone provider.

Dashboard

Turn information into knowledge

Once the phones connect to the network when brought home at the end of the shift they send the data back to the server. The data is cleaned, aligned, and pushed to a Google Fusion Table which serves as both a database, and a user interface for the client to manually edit data if necessary. From this data customized dashboards display insights and benchmarks to the client that allow them to inform their decision making process, and allow them to monitor the impacts of the decisions they make.
